# 交付阶段路由器

## 触发条件

执行阶段完成，产出物已存在。

## 路由逻辑（二维决策）

### 第一维：判断交付类型

| 类型 | 接收方 | 典型场景 |
|------|--------|---------|
| 外部交付 | 客户/老板/用户 | 交报告、发邮件、上线产品 |
| 内部交付 | 下一步/明天的自己 | 保存工作状态、写交接文档 |
| 认知交付 | 别人的大脑 | 分享思想/框架/决策逻辑 |

### 第二维：判断产出形态

| 产出形态 | 检查重点 |
|---------|---------|
| 文档 | 格式正确、内容齐全、符合标准 |
| PPT | 叙事流畅、视觉清晰、关键信息突出 |
| 代码库 | 可运行、有文档、通过测试 |
| 认知 | 逻辑链完整、可复述、可独立应用 |

### 输出：进入对应 checklists/<type>.md 的检查清单
