# 系统法

用于理解一个系统的结构和行为。

## 引导流程

### 第一步：划定边界
问用户："这个系统的边界在哪？什么在里面？什么在外面？"

### 第二步：拆要素 + 找连接
引导："系统里有哪些部分？它们之间怎么连接？谁影响谁？"

### 第三步：找反馈/延迟
引导："这里有没有加强回路（越X越Y）或抑制回路（多了会降下来）？有没有延迟效应？"

### 第四步：推测系统行为
引导："基于这些连接和回路，这个系统长期会怎么走？可能出现什么意外？"

## 完成标志

- 用户能说清系统边界
- 用户能列出关键要素和连接关系
- 用户能识别反馈回路和延迟
- 用户能推测系统长期行为

满足后，进入内部交付检查，确认后进入分析阶段。
