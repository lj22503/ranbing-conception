# 理解阶段路由器

## 触发条件

用户表达"想理解/搞懂/弄明白/认识 XXX"时，进入理解阶段。

## 路由逻辑（二维决策）

### 第一维：判断目标类型

| 用户表述特征 | 目标类型 | 对应方法 |
|-------------|---------|---------|
| "XX是什么意思" / "XX是怎么定义的" | 概念 | 概念法 |
| "XX是什么情况" / "发生了XX" | 情境 | 情境法 |
| "XX这个人" / "XX怎么想的" | 人物 | 人物法 |
| "XX这个系统" / "XX之间怎么联动" | 系统 | 系统法 |

### 第二维：选择方法来源

1. 列出该类型可用的 **builtin** 方法
2. 查询 `registry/discovered/index.md` 中该类型的 **discovered** 工具
3. 检查本地已安装的 **custom** 工具
4. 给用户呈现选项，让用户选
