# Element Analysis

## Overview

A structural decomposition framework that breaks down complex problems into constituent elements using MECE principle (Mutually Exclusive, Collectively Exhaustive). Essential for root-cause analysis and systems thinking.

## When to Use

**Trigger conditions:**
- User says "break this down", "what are the components?", "why is this happening?"
- Complex problem with unclear structure
- Root cause analysis needed
- Understanding system composition
- phrases like "要素", "拆解", "组成部分", "结构"

**When NOT to use:**
- When problem is already clearly atomic (single simple issue)
- When user needs quick decision, not deep analysis
- When emotional support is primary need (use understand stage)

## Core Framework

### The MECE Principle

| Term | Meaning | Application |
|------|---------|-------------|
| **Mutually Exclusive** | No overlap between elements | Each element is distinct |
| **Collectively Exhaustive** | All elements together cover the whole | Complete coverage, no gaps |

### Step 1: Define Boundaries

" What is the boundary of what you're analyzing? What is included? What is excluded?"

### Step 2: First-Level Decomposition

"From the highest level, what are the major components?"
- Check: Are they mutually exclusive?
- Check: Do they cover the whole?

### Step 3: Drill Down

For each key component: "Can this be further decomposed?"
- Continue until reaching actionable granularity

### Step 4: Identify Key Elements

"Among these elements, which are critical? Which are details?"
- Apply 80/20 principle
- Distinguish root causes from symptoms

### Step 5: Reconstitute

"Looking at the decomposition, what do you see that you didn't notice before?"

## Quick Reference

| Stage | Question | Check |
|-------|----------|-------|
| Boundary | "What's in and out?" | Scope clarity |
| First level | "Major components?" | MECE check |
| Drill down | "Sub-components?" | Depth adequate? |
| Key elements | "Critical vs detail?" | Pareto apply |
| Reconstitute | "New insight?" | Synthesis |

## Implementation Steps

1. **Set scope** — Clearly define boundaries
2. **Initial decomposition** — Break into 3-7 major parts
3. **Validate MECE** — Check for overlap and gaps
4. **Iterate** — Decompose each part as needed
5. **Prioritize** — Identify critical elements
6. **Synthesize** — Derive new understanding from structure

## Common Mistakes

| Mistake | Correction |
|---------|------------|
| Overlapping categories | Revise until mutually exclusive |
| Incomplete decomposition | Check against original boundary |
| Stopping too early | Drill until elements are actionable |
| No prioritization | Not all elements matter equally |

## Completion Criteria

- [ ] Boundaries clearly defined
- [ ] Decomposition is MECE at each level
- [ ] Key elements identified and marked
- [ ] User gains new insight from structure
- [ ] Root causes distinguishable from symptoms
