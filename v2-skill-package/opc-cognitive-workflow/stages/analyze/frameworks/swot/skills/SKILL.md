# SWOT Analysis

## Overview

A strategic planning framework that evaluates internal Strengths and Weaknesses against external Opportunities and Threats. Helps users make informed strategic decisions by surfacing hidden factors.

## When to Use

**Trigger conditions:**
- User mentions "strategy", "competitive", "positioning", "risk assessment"
- User is evaluating a business decision with multiple stakeholder considerations
- User says "should I do X or Y" where both have complex tradeoffs
- When analyzing market entry, product launch, or partnership decisions

**When NOT to use:**
- When the problem is purely operational (use execute methods instead)
- When user needs deep technical analysis (use first-principles instead)
- When the scope is too narrow (single constraint, no strategic dimension)

## Core Framework

### Step 1: Fill the Four Quadrants

Guide user to populate each quadrant:

| Quadrant | Question to Ask | Focus |
|----------|----------------|-------|
| **S - Strengths** | "What advantages do you have?" | Internal, positive, controllable |
| **W - Weaknesses** | "What areas need improvement?" | Internal, negative, controllable |
| **O - Opportunities** | "What external trends favor you?" | External, positive, partially controllable |
| **T - Threats** | "What external challenges exist?" | External, negative, partially controllable |

### Step 2: Cross-Analysis

Generate strategic options from quadrant combinations:

| Combination | Strategic Question |
|-------------|------------------|
| **SO** | "How can you use strengths to capture opportunities?" |
| **WO** | "How can you use opportunities to overcome weaknesses?" |
| **ST** | "How can you use strengths to defend against threats?" |
| **WT** | "What defensive moves minimize weaknesses and threats?" |

### Step 3: Synthesis

Help user derive actionable conclusions from cross-analysis.

## Quick Reference

| Element | Description |
|---------|-------------|
| Strengths | Internal advantages (resources, capabilities, assets) |
| Weaknesses | Internal gaps (lack of resources, skills, processes) |
| Opportunities | External favorable conditions (market trends, regulatory changes) |
| Threats | External challenges (competition, market shifts, risks) |

## Implementation Steps

1. **Prepare the matrix** — Draw 2x2 grid, explain each quadrant to user
2. **Fill S & W** — Ask internal questions first (controllable factors)
3. **Fill O & T** — Ask external questions (market/environment factors)
4. **Cross-analysis** — Generate 4 strategic options from combinations
5. **Prioritize** — User selects based on feasibility and impact
6. **Document conclusion** — Clear action direction from synthesis

## Common Mistakes

| Mistake | Correction |
|---------|------------|
| Listing activities as strengths | Strengths must be enduring advantages, not just actions |
| Confusing S with O | S is internal/controllable, O is external/contextual |
| Skipping cross-analysis | The value is in the combinations, not the quadrants |
| Generic statements | Each item must be specific and actionable |
| No prioritization | Cross-analysis produces options; user must choose |

## Completion Criteria

- [ ] All four quadrants filled with specific items
- [ ] Cross-analysis completed for all four combinations
- [ ] User has derived and articulated strategic conclusions
- [ ] Next action identified (if in execution flow)
