# First Principles Thinking

## Overview

A critical thinking framework that deconstructs problems to their fundamental axioms and rebuilds solutions from scratch. Effective for innovation, problem-solving, and challenging conventional wisdom.

## When to Use

**Trigger conditions:**
- User says "but why do we do it this way?", "is this assumption correct?"
- Problem seems stuck despite multiple attempts
- Conventional solutions have failed
- User wants to innovate rather than iterate
- phrases like "从根基", "重新思考", "推倒重来", "假设是什么"

**When NOT to use:**
- When user needs quick execution (too time-intensive)
- When the problem is clearly operational, not conceptual
- When existing solutions are working adequately
- When user lacks domain expertise to question assumptions

## Core Framework

### Step 1: Identify Existing Assumptions

Ask user to list all assumptions about the problem:
- "What do you believe to be true about this situation?"
- "What have you accepted as fact without questioning?"

### Step 2: Question Each Assumption

For each assumption, probe:
- "Is this definitely true?"
- "What evidence supports this?"
- "Are there counterexamples?"
- "What if this assumption were wrong?"

### Step 3: Rebuild from Foundation

If assumptions collapse:
- "Given this is false, what must be true instead?"
- "From these fundamental constraints, what solution emerges?"
- "What would you do if you had no preconceptions?"

## Quick Reference

| Stage | Question | Goal |
|-------|----------|------|
| Identify assumptions | "What do you assume to be true?" | List all beliefs |
| Challenge assumptions | "Is this always true? What if it's wrong?" | Find weak foundations |
| Rebuild | "From these facts, what follows?" | Generate new solutions |

## Implementation Steps

1. **Surface assumptions** — Ask user to verbalize all implicit beliefs
2. **Categorize** — Separate assumptions into: facts, beliefs, conventions
3. **Challenge each** — Apply Socratic questioning to every assumption
4. **Preserve valid** — Keep only assumptions that survive scrutiny
5. **Rebuild** — Construct new solution from surviving axioms
6. **Validate** — Check new solution addresses original problem

## Common Mistakes

| Mistake | Correction |
|---------|------------|
| Skipping assumption identification | Cannot challenge without first articulating |
| Treating conventions as facts | "Industry standard" ≠ fundamental truth |
| Abandoning working solutions | First principles is for stuck problems, not all problems |
| Mistaking skepticism for wisdom | Questioning everything = analysis paralysis |

## Completion Criteria

- [ ] All key assumptions explicitly stated
- [ ] Each assumption questioned with evidence
- [ ] Invalid assumptions identified and discarded
- [ ] New solution rebuilt from valid foundations
- [ ] User articulates new understanding or approach
