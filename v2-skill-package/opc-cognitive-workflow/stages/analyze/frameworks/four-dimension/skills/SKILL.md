# Four-Dimension Evaluation

## Overview

A critical evaluation framework that assesses claims across four dimensions: Source, Logic, Assumptions, and Verifiability. Helps distinguish credible information from unreliable sources.

## When to Use

**Trigger conditions:**
- User says "should I believe this?", "is this source reliable?", "how do I evaluate X?"
- User is considering adopting someone else's opinion or data
- User needs to validate claims before making decisions
- phrases like "可信吗", "怎么判断", "谁说的对", "靠不靠谱"

**When NOT to use:**
- When user already trusts the source completely
- When evaluating purely technical merit (use element analysis)
- When emotional acceptance is the goal, not rational evaluation

## Core Framework

### The Four Dimensions

| Dimension | Core Question | What to Examine |
|-----------|--------------|----------------|
| **Source** | Who said it? How credible are they? | Expertise, track record, potential bias |
| **Logic** | Does the reasoning hold? | Argument structure, inference quality, fallacies |
| **Assumptions** | What is this based on? Are premises valid? | Hidden premises, stated conditions |
| **Verifiability** | Can this be proven? | Data, evidence, reproducibility |

### Step 1: Identify the Object

"What exactly are you evaluating? A claim? A proposal? Information from a source?"

### Step 2: Score Each Dimension

Rate each dimension 1-5 with reasoning:

| Score | Meaning |
|-------|---------|
| 5 | Fully credible, strong evidence |
| 4 | Mostly credible, minor concerns |
| 3 | Moderate credibility, significant gaps |
| 2 | Weak credibility, major issues |
| 1 | Not credible |

### Step 3: Synthesis

"Given these four dimension scores, what is your overall judgment?"

## Quick Reference

| Dimension | Red Flags |
|-----------|----------|
| Source | Anonymous, vested interest, no track record |
| Logic | Non-sequitur, circular reasoning, false dilemma |
| Assumptions | Unstated, unfalsifiable, contradicting evidence |
| Verifiability | No data, cherry-picked, cannot be replicated |

## Implementation Steps

1. **Identify target** — Clarify what is being evaluated
2. **Assess Source** — Check credibility indicators
3. **Analyze Logic** — Map argument structure
4. **Surface Assumptions** — Identify hidden premises
5. **Check Verifiability** — Look for evidence quality
6. **Synthesize** — Form overall judgment

## Common Mistakes

| Mistake | Correction |
|---------|------------|
| Trusting authority over evidence | Source credibility ≠ claim validity |
| Ignoring hidden assumptions | Unstated assumptions can invalidate conclusions |
| Demanding certainty | Verifiability exists on a spectrum |
| Scoring without reasoning | Each score needs justification |

## Completion Criteria

- [ ] Object clearly identified and framed
- [ ] Each dimension scored with explicit reasoning
- [ ] Red flags identified and noted
- [ ] User forms and articulates synthesis judgment
