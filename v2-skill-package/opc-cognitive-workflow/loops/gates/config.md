# 纪律层配置

## 门禁强度等级

| 等级 | 行为 | 触发后果 |
|------|------|---------|
| **HARD** | 强制拦截，不可绕过 | 必须满足才能进入下一步 |
| **SOFT** | 警告，可选择继续 | 提醒但用户可决定是否继续 |

## 门禁配置表

| 门禁名称 | 源文件 | 默认强度 | 适用场景 | 配置选项 |
|---------|--------|---------|---------|---------|
| 阶段完成检查 | internal-handoff.md | HARD | 阶段间交接 | HARD / SOFT |
| 质量门禁 | quality-gate.md | HARD | 交付前 | HARD / SOFT |
| 退化检测 | degradation-detection.md | SOFT | 全程监控 | SOFT |
| 执行检查点 | execute-checkpoint.md | SOFT | 执行每步 | SOFT |

## 强度调整规则

### 何时升级为 HARD

当满足以下任一条件时，将 SOFT 升级为 HARD：
- 用户反馈"之前的检查没有拦住问题"
- 使用场景为高风险决策（金融、医疗、法律）
- 用户明确要求严格模式

### 何时降级为 SOFT

当满足以下任一条件时，将 HARD 降级为 SOFT：
- 用户反馈"检查过于频繁，打断思考流"
- 使用场景为创意发散、头脑风暴
- 用户明确要求宽松模式

## Anti-Rationalization 规则

### 通用红线

以下行为被严格禁止：

| 禁止行为 | 正确做法 |
|---------|---------|
| 替用户做判断 | 提供框架和选项，让用户选择 |
| 跳过阶段 | 必须完成每个阶段的交付检查 |
| 只给一个选项 | 至少提供2个选项，除非只有唯一可能 |
| 跳过产出说明 | 每个阶段必须明确产出物 |
| 用户未确认就推进 | 先问"可以进入下一步了吗" |

### 退化信号处理

| 信号 | 第1次 | 第2次 | 第3次 |
|------|-------|-------|-------|
| 替用户做判断 | 提醒 | 暂停+说明 | 回到理解阶段 |
| 跳过阶段 | 提醒 | 暂停+说明 | 强制补充 |
| 只给单一选项 | 提醒 | 暂停+补充 | 强制补充2个 |
| 无产出说明 | 提醒 | 暂停+说明 | 回到本阶段重建 |

## 激活状态

默认所有门禁为激活状态。可以通过以下方式修改：

```markdown
// 在对话开始时声明
[使用严格模式] - 所有SOFT升级为HARD
[使用宽松模式] - 所有HARD降级为SOFT
[使用创意模式] - 退化检测暂停，质量门降为SOFT
```
