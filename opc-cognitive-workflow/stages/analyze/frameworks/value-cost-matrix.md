# 分析框架：价值-成本矩阵

## 适用问题

用户面临多个选项，需要排优先级、做选择。

## 核心模型

```
选项 → 评估价值（对我多重要）→ 评估成本（我要付出什么）→ 填入矩阵 → 得优先级
```

## 引导流程

### Step 1：穷举选项

> "现在有哪些选项？先把所有选项都列出来，不做筛选。"

### Step 2：逐项评估价值

对每个选项问：

> "如果做成这件事，对你的最终目标有多大贡献？"
> "这个贡献是锦上添花，还是至关重要？"

记录：价值高/中/低。

### Step 3：逐项评估成本

对每个选项问：

> "做这件事，你需要付出什么？（时间/金钱/精力/机会成本）"
> "这个代价你觉得值吗？"

记录：成本高/中/低。

### Step 4：填入矩阵并得出结论

| | 价值高 | 价值中 | 价值低 |
|---|---|---|---|
| **成本低** | 优先做 ✅ | 可以做 | 放弃 |
| **成本中** | 想办法降低门槛 | 往后排 | 放弃 |
| **成本高** | 想办法降低门槛 | 放弃 | 放弃 |

> "按这个矩阵，你的优先级顺序是…？"

## 你的角色

你只负责引导和框架搭建，**结论由用户自己得出**。如果用户的结论和矩阵不符，追问一句："你的判断和矩阵不一致，是有什么矩阵没考虑到的因素吗？"
